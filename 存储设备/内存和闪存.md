# NAND闪存

比较有代表性的是DRAM(内存)和NAND(闪存)，这两个词近半年频繁的在我耳边出现，牛逼轰轰的。害得我不得不去啃一啃其差异。

由于这两种存储方式都是基于MOS管开发的，因此我们需要先简单回顾下MOS管的工作原理。

简单来讲就是给栅极(G)加电压，会在栅极(G)下方形成一个电子沟道，这个电子沟道可以使漏极(D)、源极(S)导通；因此便可以通过控制栅极(G)电压来实现MOS管漏、源导通与断开。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302184317894.png)


闪存(Flash)读写方式：

说到这里可能有些记性好的老粉会问我为什么开始炒冷饭了！确实，在去年差不多这个时候，基于《图解-芯片技术》那本书确实写过一篇flash闪存的读写原理。今天这篇文章原计划是只写内存，闪存贴个链接就好了。但恰巧在前几天，看到一篇更加通俗易懂的闪存读写原理介绍，于是乎，我就手痒了！

先看看闪存的结构介绍：

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302184643285.png)

可以看到，和普通的MOS管的差异，就是在栅极的绝缘层之间增加了一层浮栅。浮栅与衬底之间被一层叫“隧道氧化层”的绝缘层隔开。

与上面MOS管原理介绍的一样，当给栅极加正电压时，由于电场力作用，会将衬底中的电子吸附至栅极下方，形成一条电子沟道。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302185148419.png)

当栅极电压增大，大到电场强度大于1000万V/cm时，电子会穿过隧道氧化层到达浮栅中(这种方式被网友通俗的称为大力出奇迹)。因此隧道氧化层在确保绝缘性的前提下，越薄电子越容易穿过。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302185216906.png)

当电压撤除时，由于浮栅上下均被绝缘介质包裹，因此在没有外界电场力作用的情况下，泄露很慢，可以保存数十年。。。**这也是闪存属于非易失性存储器（NVM）的原因——断电后数据不会丢失。****因此广泛应用于固态硬盘（SSD）、U盘等存储设备中。**。
![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302190510198.png)
这就是闪存的写操作。那么如何进行擦除呢？

很简单，反向操作：
![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302190534169.png)

衬底接正压，栅极接负压，压差加到足够大，浮栅中的电子则会从隧道氧化层重新穿回衬底，怎么吃进去的，怎么给我吐出来。

写入和擦除，电子都需要穿过隧道氧化层，次数多了，就会导致隧道氧化层的绝缘性能变差，这也就是为什么flash芯片寿命介绍中都会有一个最高可支持的擦除次数。

再聊聊读操作。

当浮栅中没有电子时，栅极加正电压，在栅极下方形成导电沟道，漏、源导通，读数端口可以检测到VCC的电压值，置逻辑1(下图左)；

当浮栅中有电子时，电子会抵消掉栅极与衬底的压差，源、漏无法形成导电沟道，读数端口无法读取到VCC的电压值，置逻辑0(下图右)。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260302190604020.png)

# DRAM内存

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260303213900522.png)
MOS管端与普通的并无差异(与闪存相比栅极没有浮栅)，源极与地之间，接了一个电容器。

写入操作：MOS管栅极接正压，衬底接负压。栅极下方形成导电沟道，漏极-->源极-->电容-->地 形成的回路导通。由于漏极接正压，电流通过导通的沟道给电容器充电，电荷便**存储在电容器的极板**上。电容充满电荷后，撤除栅极电压，导电沟道消失。电容会保持该充电状态。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260303213918762.png)

当然停止充电后电容器中的电荷因漏电流的存在会慢慢逃逸(不能像闪存中的浮栅一样将电荷长久锁住)，因此需要定期进行通电刷新数据。这也是其被称为易失存储器的原因——断电后刷新操作停止，电荷逃逸完则数据消失。

读取操作：栅极接正电压，导电沟道形成，源、漏极导通。

当电容器中有电荷时，电容器放电，漏极则可检测到电信号，即为逻辑1；

当电容器中无电荷时，漏极则不能检测到电信号，即为逻辑0.

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260303213941285.png)

擦除操作：栅极加正电压使源、漏导通，漏极接地电容则会进行快速放电，数据消失。实际上每一次读取数据时，电容都会放电——这就是DRAM的**破坏性读出**特性。因此，读操作完成后，内存控制器会立即将读出的数据重新写回电容（称为**回写**），否则数据就会丢失。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260303213957388.png)

# 为什么DRAM芯片比NAND芯片读写速度更快？

接触存储芯片之前，只知道那玩意就是U盘、固态硬盘等里面用的芯片。以往在选择购买的时候，也只知道存储容量大的贵，小的便宜。后来出现了USB2.0/3.0，知道了接口协议会影响拷贝速度，但对于什么是接口协议也是稀里糊涂。再往后，慢慢接触到DDR、DRAM、NAND这些概念，才明白芯片本身也分很多种——有些用来做内存，有些用来做固态硬盘，它们虽然都叫存储芯片，干的却是完全不同的活儿。这时候我才意识到，要搞清楚这些问题，还得回到计算机系统的基本工作原理上去。。。

于是查阅了一些资料，知道了计算机在运行大型软件的时候需要先从硬盘中拿出数据放入内存中，再以内存为临时数据存放载体，进行逻辑运算，最终的运算结果放回硬盘中进行长期保存。内存的作用为临时存放数据，特点是数据读写速度很快，但是不能做长期存放，断电会丢失数据。而硬盘读写速度远低于内存，但优势是可以长期存储数据。

那么为什么内存芯片读写速度快，闪存芯片(固态硬盘)读写速度慢，我们以DRAM(内存芯片)和NAND(闪存芯片)为例探讨下。

从数据存储原理来讲，DRAM依赖的是MOS管串联电容的充放电，而NAND依赖的是电子在浮栅中的存储。电容的充放电速度在纳秒级，而电子穿越隧道氧化层到达浮栅的速度在微秒级。这就导致了从底层逻辑上DRAM远快于NAND。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260330235312897.png)


什么原因导致这个数量级差异？对于DRAM芯片，MOS管导通后相当于一个导体，电容充放电时电子运动是在导体中进行的。而NAND芯片电子需要穿越绝缘材质隧道氧化层才能写入或擦除，就像得了尿结石一样。。。我的一个朋友告诉我说很疼！

这是导致两种芯片速度差异的底层原因，近期了解了两种芯片的内部结构和工作方式后，明白了还有读写逻辑带来的差异。

我们在上一篇讲过，NAND芯片内部单元阵列是坐落在**一个个独立的衬底阱区上的，每个块（或几个块）拥有自己独立的衬底阱，**擦除数据需要给**目标块的阱加很大的电压（字线接0V），形成强电场将电子从浮栅拉出**，因此擦除最小单位是块。要想修改其中某页的数据，需要将**该页所在块的所有有效数据**先存储在内部寄存器中，再对整块数据进行擦除，然后再重新将所有数据写回去，这么一套流程下来（读→擦除→写），单从流程上就很耗时了。

![](https://raw.githubusercontent.com/LeroyK111/pictureBed/master/20260330235406171.png)

反之DRAM的衬底只是用来配合提供沟道导通电子的，就没有这个烦恼。**（不过DRAM也有自己的麻烦——需要不断刷新，否则电容里的电荷会慢慢漏掉）**。DRAM可以利用阵列结构以及列寻址来精确控制每个单元栅极导通，然后直接对电容器进行数据读、写。

因此NAND芯片由于这些特性，天生就是干仓库的料。也是基于这些原因，聪明的大胡子科学家们对存储进行分类，各司其职。一个管速度，一个管容量。NAND芯片将位线上的各单元进行串联缩小独立单元面积，输出数据时先存入寄存器，再以串行方式进行输出以减小接口数量也是为了减小面积，从而获得容量最大化。DRAM芯片则利用并行输出的方式，牺牲面积提升数据读写速度——**串行接口引脚少但速度慢，并行接口速度快但占面积**，这也是接口设计上的一个取舍。不过这块还没来得及搞得很清楚，今天先这么着了。。。

所以现在回头看，DRAM和NAND一个快一个慢，根本原因确实在存储原理上——电容充放电就是比电子隧穿快。但结构设计、操作流程、接口方式这些因素又把差距进一步拉大了。
